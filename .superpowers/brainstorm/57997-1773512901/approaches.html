Three Implementation Decisions

The PRD covers most everything. A few architectural choices remain — pick your preference for each.

1. Pi Digits Source

How should we get the 1M digit file?

A

Node.js download script

Fetch from pi2e.ch at build time, validate first 20 digits, save to public/pi-digits.json. Reliable, ~1MB file. Requires network during setup.

B

Generate via BBP algorithm

Use a pure-JS pi computation library to generate digits locally — no network needed. Slower to generate (~30s) but fully self-contained.

2. Digit Stream Animation Engine

How should the scanning animation be driven?

A

✓ Recommended — requestAnimationFrame

Precise 60fps control. Speed ramp logic is straightforward JS. Matches the PRD spec exactly. GPU-accelerated via transform updates.

B

CSS keyframe animations

Simpler code. But harder to control speed ramping dynamically, and near-miss flashes require JS anyway. Not recommended for this use case.

3. Vercel Deployment

Should deployment be part of this session?

A

Yes — build + deploy to Vercel

After npm run build passes, run the Vercel deploy skill. Live URL included in the share button text.

B

No — local build only

Finish with npm run build && npm start. Deploy separately later. Share button can use a placeholder URL.
